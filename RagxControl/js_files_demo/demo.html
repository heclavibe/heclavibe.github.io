Text Alignment

align: 'left' | 'right' | 'justify' | 'center'

The measure is the length of a line of type. To a reader's eye, long or short lines can be tiring and distracting. Setting a block of text with a good rag creates a pleasing visual rhythm that guides the eye naturally from line to line without interruption or distraction.

ragxcontrol({ selector: 'p', align: 'justify' });

Responsive Breakpoints

maxCharsPerLine per viewport

Resize this window to see how RagXControl constrains the line length differently on mobile, tablet, and desktop viewports. The ideal measure for body text is between 45 and 75 characters — this text dynamically adapts its maximum width using ch units.

simulate:
ragxcontrol({ breakpoints: { mobile: { maxWidth: 767, maxCharsPerLine: 40 }, tablet: { maxWidth: 1023, maxCharsPerLine: 58 }, desktop: { maxWidth: Infinity, maxCharsPerLine: 72 } } });

Hyphenation

Automatic soft-hyphen insertion

Extraordinarily long words like pneumonoultramicroscopicsilicovolcanoconiosis and antidisestablishmentarianism can disrupt the rag significantly. RagXControl inserts soft hyphens at computed syllable boundaries, letting the browser break these words elegantly when needed.

| narrow container:
ragxcontrol({ hyphenation: { enabled: true, minWordLength: 6, minCharsBeforeBreak: 3, minCharsAfterBreak: 3 } });

Widows & Orphans

Glue final words together with &nbsp;

Without widow control a single short word can end up stranded alone on the last line of a paragraph which looks quite bad.

| glue last:
ragxcontrol({ widows: { enabled: true, minWords: 2 } });

The shape of a rag is the visual silhouette created by the line endings of a paragraph. Typography benefits from deliberate decisions about this silhouette. RagXControl provides multiple shape algorithms so that designers can move far beyond the browser default with full control.

intensity:
ragxcontrol({ shape: 'wave', shapeIntensity: 0.7 });

Small Words & Prepositions

Prevent short words orphaning at line-start

A good rag avoids leaving articles like "a", "an", "the" and prepositions like "in", "of", "to" stranded alone at the end of a line before the next word wraps below it.

ragxcontrol({ smallWords: { enabled: true, maxLength: 3 }, prepositions: { enabled: true } });

The finest typographic details are often invisible to the casual reader — but their absence is immediately felt. A widow stranded on the last line of a column, a preposition hanging at the end of a sentence before its object wraps, or an extraordinarily long word that ruptures the edge of a paragraph: these are the small disruptions that undermine readability.

RagXControl operates on the conviction that the web need not be a second-class typographic environment. Every rule that a book compositor applies when setting body text — controlling the measure, managing the rag, hyphenating with care — can be approximated in the browser with a few kilobytes of thoughtful JavaScript.

The measure of a good paragraph is not only the number of characters per line, but the rhythm of the line endings taken as a whole. A well-set rag flows in gentle, irregular steps — never a sudden hole, never a line conspicuously longer than its neighbours. This is the difference between a page that reads and one that merely displays.

// Full prose setup const rm = ragxcontrol({ selector: 'article p', align: 'left', breakpoints: { mobile: { maxWidth: 767, maxCharsPerLine: 42 }, tablet: { maxWidth: 1023, maxCharsPerLine: 62 }, desktop: { maxWidth: Infinity, maxCharsPerLine: 75 } }, hyphenation: { enabled: true, minWordLength: 7 }, widows: { enabled: true, minWords: 2 }, smallWords: { enabled: true }, prepositions: { enabled: true }, dashes: { enabled: true }, shape: 'natural', observe: true });
